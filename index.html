Competition Guidelines

Technical Requirements

  • Static Content: Pure HTML, CSS, and JavaScript only
  • No Backend: No backend dependencies or build steps
  • Static Assets: All content must be static
  • Responsive Design: Must use Flexbox/Grid
  • Performance: Optimized for fast loading
  • Deployment: Must be deployable on GitHub Pages
  • Blog Section:Ability to add and remove blog posts.
  • SEO Friendly:Should be SEO friendly.

Judging Criteria

  • Design & Aesthetics (40%): UI/UX, visual appeal, animations
  • Responsiveness (30%): Mobile-first approach, cross-device compatibility. Tools like Google Page Speed Insights will be used for benchmarking.
  • AI Integration (20%): Creative use of AI tools for development
  • Code Quality (10%): Clean, maintainable, and well-documented code

Eligibility

  • Students: Currently enrolled in any Pakistani university
  • Team Size: 2, 4 members per team (max 4). You can also participate solo but there are bonus marks for team submissions.
  • Experience: Open to all skill levels
